# 自然语言识别与交互

自然语言处理 (Natural Language Processing, NLP) 技术使驾驶员能够通过语音与车辆交互，实现语音控制、信息查询和智能对话等功能。随着大语言模型的发展，车载语音交互正从简单的指令识别向自然对话演进。


## 语音识别 (ASR)

自动语音识别 (Automatic Speech Recognition, ASR) 将语音信号转换为文本，是语音交互的第一步。

处理流程：

1. **语音活动检测 (VAD)**：检测语音起止点，过滤静音和噪声
2. **特征提取**：提取梅尔频率倒谱系数 (MFCC) 或 Fbank 等声学特征
3. **声学模型**：将特征映射为音素或字符序列，主流方案有 CTC、Attention、Transducer
4. **语言模型**：结合上下文纠正识别结果，提高准确率

车载 ASR 的特殊挑战：

- 车内噪声（发动机、风噪、路噪）导致信噪比低
- 多人对话时需要声源定位和说话人分离
- 方言和口音的多样性
- 需要在车载芯片上离线运行，保证无网络时仍可用


## 自然语言理解 (NLU)

自然语言理解将 ASR 输出的文本转化为结构化的语义表示。核心任务包括：

- **意图识别 (Intent Classification)**：判断用户意图，如"导航到公司"→导航意图
- **槽位填充 (Slot Filling)**：提取关键实体，如"目的地=公司"
- **多轮对话管理**：跟踪对话状态，处理指代消回、上下文补全等

车载场景中常见意图类型：

| 意图类别 | 示例 | 对应操作 |
| --- | --- | --- |
| 导航控制 | "导航去最近的加油站" | 设置导航目的地 |
| 车辆控制 | "打开空调，调到 24 度" | 调节车载设备 |
| 多媒体控制 | "播放周杰伦的歌" | 控制音乐播放 |
| 信息查询 | "今天天气怎么样" | 联网查询并播报 |
| 驾驶相关 | "还有多远到达" | 查询导航信息 |


## 对话系统

车载对话系统需要将理解结果转化为实际操作：

- **任务型对话**：基于预定义的对话流程完成特定任务（如设置导航），采用对话状态跟踪 (DST) + 策略 + 自然语言生成 (NLG)
- **问答系统**：回答驾驶相关问题，如路况、车辆状态等
- **闲聊与情感交互**：缓解驾驶疲劳，提升用户体验

近年来基于大语言模型 (LLM) 的车载助手（如奔驰 MBUX、理想同学、小鹏语音）正在取代传统的流水线方案，实现更自然的多轮对话。


## 中文语音交互的挑战

- **同音字歧义**：中文存在大量同音字，需要结合上下文消歧
- **分词困难**：中文缺乏明确的词边界，分词质量直接影响 NLU
- **语气词与省略**：口语化表达中频繁出现"嗯"、"那个"等语气词和主语省略
- **中英混杂**：导航地点、歌曲名等常包含英文，需跨语言识别
